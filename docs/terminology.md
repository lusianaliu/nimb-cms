# Nimb CMS Terminology (Phase 0 Draft)
Core: the minimal platform runtime and extension contracts.
Plugin: a modular package that adds behavior, integrations, or workflows.
Theme: a presentation layer responsible for rendering and styling.
Block: a structured content unit independent of visual layout.
Config: environment-specific settings for system behavior.
Domain-Agnostic: reusable logic not tied to a specific business vertical.
Phase 0: repository identity and foundational vocabulary setup.
